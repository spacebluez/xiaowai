# 小歪 Xiaowai 前端联调说明

## 1. 联调目标

本文档用于指导 `xiaowai-web` 与 `xiaowai-backend` 进行真实接口联调，重点覆盖：

- 登录注册
- 登录态保持
- 用户资料读取与更新
- 头像上传与展示
- AI 对话
- `fetch` / `axios` 调用示例

---

## 2. 后端基础信息

- 后端接口前缀：`/api/v1`
- 本地开发地址示例：`http://localhost:8080/api/v1`
- 鉴权方式：`JWT + HttpOnly Cookie`

### 重要说明
登录成功后，后端会把 `token` 写入 Cookie。后续所有需要登录的请求，前端必须携带 Cookie，否则会被判定为未登录。

---

## 3. 必须处理的联调规则

### 3.1 所有鉴权接口都要带凭证

#### fetch

```ts
fetch(url, {
  method: 'GET',
  credentials: 'include'
})
```

#### axios

```ts
axios.get(url, {
  withCredentials: true
})
```

### 3.2 鉴权失败返回不完全统一
后端有两种常见失败格式：

```json
{
  "code": 401,
  "msg": "未授权，请重新登录",
  "data": null
}
```

或：

```json
{
  "msg": "请先登录"
}
```

因此前端统一错误提示建议写成：

- 优先读 `response.data.msg`
- 如果没有，显示通用错误文案

### 3.3 头像预览接口返回的是图片流
以下接口不是 JSON：

- `GET /api/v1/user/:id/avatar`

应直接用于 `<img src>`，不要 `res.json()`。

如果前端使用 Vite，建议增加：


## 5. 推荐封装方式

## 5.1 fetch 版基础封装

```ts
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL;

export async function request<T = any>(path: string, options: RequestInit = {}) {
  const response = await fetch(`${API_BASE_URL}${path}`, {
    credentials: 'include',
    headers: {
      'Content-Type': 'application/json',
      ...(options.headers || {}),
    },
    ...options,
  });

  const contentType = response.headers.get('content-type') || '';
  const data = contentType.includes('application/json')
    ? await response.json()
    : null;

  if (!response.ok || (data && data.code && data.code !== 0)) {
    const message = data?.msg || `请求失败: ${response.status}`;
    throw new Error(message);
  }

  return data as T;
}
```

### 上传文件时注意
上传文件不要手动设置 `Content-Type`，浏览器会自动生成 multipart boundary。

```ts
export async function upload(path: string, formData: FormData) {
  const response = await fetch(`${API_BASE_URL}${path}`, {
    method: 'PUT',
    credentials: 'include',
    body: formData,
  });

  const data = await response.json();
  if (!response.ok || data.code !== 0) {
    throw new Error(data?.msg || '上传失败');
  }

  return data;
}
```

## 5.2 axios 版基础封装

```ts
import axios from 'axios';

export const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  withCredentials: true,
  headers: {
    'Content-Type': 'application/json',
  },
});

http.interceptors.response.use(
  (response) => {
    const data = response.data;
    if (typeof data?.code !== 'undefined' && data.code !== 0) {
      return Promise.reject(new Error(data.msg || '请求失败'));
    }
    return response;
  },
  (error) => {
    const message = error?.response?.data?.msg || error.message || '网络异常';
    return Promise.reject(new Error(message));
  }
);
```

后端注册参数：

- `email`
- `code`
- `username`
- `password`
- `phone`（可选）

### fetch 示例

```ts
await request('/user/register', {
  method: 'POST',
  body: JSON.stringify({
    email: 'user@example.com',
    code: '123456',
    username: 'xiaowai001',
    password: '12345678',
    phone: '13800138000',
  }),
});
```

### axios 示例

```ts
await http.post('/user/register', {
  email: 'user@example.com',
  code: '123456',
  username: 'xiaowai001',
  password: '12345678',
  phone: '13800138000',
});
```

### 注册页改造建议
当前 `Register.tsx` 还缺少：

- 验证码输入框
- 获取验证码按钮
- 手机号输入框（可选）
- 对接真实接口逻辑

### 推荐提交流程
1. 校验邮箱格式
2. 校验验证码长度 6 位
3. 校验密码和确认密码一致
4. 调用注册接口
5. 成功后跳转登录页

## 6.3 登录

后端登录使用的是：

- `username`
- `password`

### fetch 示例

```ts
const res = await request<{ code: number; msg: string; data: { token: string } }>(
  '/user/login',
  {
    method: 'POST',
    body: JSON.stringify({
      username: 'xiaowai001',
      password: '12345678',
    }),
  }
);

console.log(res.data.token);
```

### axios 示例

```ts
const res = await http.post('/user/login', {
  username: 'xiaowai001',
  password: '12345678',
});

console.log(res.data.data.token);
```

### 登录页改造建议
当前 `Login.tsx` 页面字段是：

- `email`
- `password`

但后端真实接口需要：

- `username`
- `password`

所以联调时建议把登录输入项改成“用户名”。

### 登录成功后的前端流程
1. 调用登录接口
2. 后端写入 Cookie
3. 前端跳转聊天页
4. 页面初始化时调用 `GET /user/profile`

### Profile 页面映射建议
后端资料字段与当前前端展示字段不完全一致，建议做映射：

| 当前页面字段 | 后端字段 | 说明 |
|---|---|---|
| name | nickname | 后端没有真实 name，建议用 nickname |
| email | 无 | 当前资料接口不返回邮箱 |
| phone | 无 | 当前资料接口不返回手机号 |
| bio | signature | 可以映射为个性签名 |
| joinDate | 无 | 后端未提供 |
| location | 无 | 后端未提供 |

因此 `Profile.tsx` 联调时建议优先展示：

- 昵称
- 性别
- 生日
- 爱好
- 个性签名
- 经验值
- 更新时间

## 6.6 上传头像

### fetch 示例

```ts
async function uploadAvatar(file: File) {
  const formData = new FormData();
  formData.append('avatar', file);

  const response = await fetch(`${import.meta.env.VITE_API_BASE_URL}/user/avatar`, {
    method: 'PUT',
    credentials: 'include',
    body: formData,
  });

  const data = await response.json();
  if (!response.ok || data.code !== 0) {
    throw new Error(data?.msg || '头像上传失败');
  }

  return data;
}
```

### axios 示例

```ts
async function uploadAvatarByAxios(file: File) {
  const formData = new FormData();
  formData.append('avatar', file);

  const res = await http.put('/user/avatar', formData, {
    headers: {
      'Content-Type': 'multipart/form-data',
    },
  });

  return res.data;
}
```

### 头像展示方式
获取到用户 `id` 后，可直接：

```tsx
<img src={`${import.meta.env.VITE_API_BASE_URL}/user/${userId}/avatar`} alt="avatar" />
```

### 注意
如果用户默认头像不是 OSS 地址，这个地址可能加载失败。前端应加 `onError` 兜底默认头像。

## 6.7 发送聊天消息

### fetch 示例

```ts
const res = await request<{
  code: number;
  msg: string;
  data: { output: string };
}>('/user/agent/chat', {
  method: 'POST',
  body: JSON.stringify({
    input: '你好，介绍一下你自己',
  }),
});

const output = res.data.output;
```

### axios 示例

```ts
const res = await http.post('/user/agent/chat', {
  input: '你好，介绍一下你自己',
});

const output = res.data.data.output;
```

### Chat 页面联调建议
当前 `Chat.tsx` 是本地模拟回复，联调时建议把 `handleSend` 改成：

1. 输入为空时直接返回
2. 先把用户消息插入本地列表
3. 设置 `isLoading = true`
4. 调用聊天接口
5. 把返回的 `output` 插入 assistant 消息
6. 失败时插入错误提示或 toast
7. 最后 `isLoading = false`

### 示例伪代码

```ts
async function handleSend() {
  if (!input.trim()) return;

  const currentInput = input;
  setMessages((prev) => [
    ...prev,
    { id: crypto.randomUUID(), role: 'user', content: currentInput },
  ]);
  setInput('');
  setIsLoading(true);

  try {
    const res = await request('/user/agent/chat', {
      method: 'POST',
      body: JSON.stringify({ input: currentInput }),
    });

    setMessages((prev) => [
      ...prev,
      { id: crypto.randomUUID(), role: 'assistant', content: res.data.output },
    ]);
  } catch (error) {
    setMessages((prev) => [
      ...prev,
      {
        id: crypto.randomUUID(),
        role: 'assistant',
        content: error instanceof Error ? error.message : '对话失败，请稍后再试',
      },
    ]);
  } finally {
    setIsLoading(false);
  }
}
```

### 第四步：资料页

- 读取资料
- 更新资料

### 第五步：聊天页
把 mock 对话改为真实请求。

## 8. 目前前后端不匹配点汇总

### 8.1 Login 页面字段不匹配
当前前端：
- email
- password

后端实际需要：
- username
- password

### 8.2 Register 页面缺少验证码能力
当前前端未包含：
- 验证码输入
- 发送验证码
- 手机号

### 8.3 Profile 页面字段结构不一致
当前前端有：
- email
- phone
- location
- bio
- joinDate

后端资料接口返回的是：
- nickname
- birthday
- avatar
- gender
- hobbies
- signature
- experience
- updatedat

### 8.4 Chat 页面包含未落地后端功能
当前页面中的以下模块暂无后端接口支撑：
- 最近对话
- 历史记录
- 智能体列表切换
- Prompt 广场
- 多模态上传
- 麦克风输入

因此联调时建议先聚焦最小闭环：
- 登录
- 获取资料
- 发送聊天
- 更新资料
- 上传头像

## 9. 最小可用联调闭环

推荐先做通以下链路：

1. 注册页发送验证码
2. 注册成功
3. 登录成功
4. 首页检查登录态
5. 聊天页发送消息
6. 资料页读取资料
7. 资料页更新资料
8. 上传并展示头像
9. 退出登录

## 10. 建议的错误提示文案

### 注册
- 邮箱格式不正确
- 验证码不能为空
- 验证码必须为 6 位
- 两次输入密码不一致
- 用户名已存在
- 邮箱已存在
- 验证码错误或已过期

### 登录
- 请输入用户名
- 请输入密码
- 用户名或密码错误
- 登录失败次数过多，请10分钟后再试

### 聊天
- 请输入消息内容
- 未登录，请先登录
- 对话失败，请稍后再试

### 资料
- 获取资料失败
- 更新资料失败
- 头像上传失败
