# 小歪 Xiaowai 后端接口文档

## 1. 基本信息

- 项目：`xiaowai-backend`
- 服务默认端口：`8080`
- 接口前缀：`/api/v1`
- 认证方式：`JWT + Cookie`
- 数据格式：`application/json`
- 文件上传：`multipart/form-data`

---

## 2. 统一响应结构

大多数接口返回统一 JSON：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

字段说明：

- `code`：业务状态码，成功为 `0`，失败通常等于 HTTP 状态码
- `msg`：提示信息
- `data`：返回数据，可能为对象、数组或 `null`

### 注意
鉴权中间件失败时，可能直接返回：

```json
{
  "msg": "请先登录"
}
```

前端联调时需要兼容这种非统一结构。

---

## 3. 认证说明

### 3.1 登录态来源
登录成功后，后端会：

1. 在响应体中返回 `token`
2. 通过 `Set-Cookie` 写入 `token` Cookie

### 3.2 受保护接口认证方式
受保护接口从 Cookie 中读取 `token`，不是从 `Authorization` 头读取。

### 3.3 前端请求要求
跨域请求时必须携带凭证：

- `fetch`：`credentials: 'include'`
- `axios`：`withCredentials: true`

### 3.4 当前允许跨域来源
- `http://localhost:5173`
- `http://47.111.12.222:5173`

## 5. 验证码模块

### 5.1 发送邮箱验证码

- 方法：`POST`
- 路径：`/api/v1/verification/send-code`
- 是否鉴权：否
- Content-Type：`application/json`

#### 请求参数

```json
{
  "email": "user@example.com"
}
```

#### 参数说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| email | string | 是 | 邮箱地址，必须符合邮箱格式 |

#### 成功响应

```json
{
  "code": 0,
  "msg": "验证码已发送，请查收邮件",
  "data": null
}
```

#### 失败响应

```json
{
  "code": 400,
  "msg": "参数错误:...",
  "data": null
}
```

```json
{
  "code": 429,
  "msg": "验证码获取操作频繁，60s后再试",
  "data": null
}
```

#### 限制规则

- 同一邮箱 60 秒内不能重复获取
- 同一邮箱每天最多 10 次
- 同一 IP 每分钟最多 5 次
- 验证码有效期 5 分钟

## 6. 用户模块

### 6.1 用户注册

- 方法：`POST`
- 路径：`/api/v1/user/register`
- 是否鉴权：否
- Content-Type：`application/json`

#### 请求参数

```json
{
  "email": "user@example.com",
  "code": "123456",
  "username": "xiaowai001",
  "password": "12345678",
  "phone": "13800138000"
}
```

#### 参数说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| email | string | 是 | 邮箱 |
| code | string | 是 | 6 位验证码 |
| username | string | 是 | 用户名，长度 2-50 |
| password | string | 是 | 密码，最少 6 位 |
| phone | string | 否 | 手机号 |

#### 成功响应

```json
{
  "code": 0,
  "msg": "注册成功",
  "data": null
}
```

#### 失败响应

```json
{
  "code": 422,
  "msg": "用户名已存在",
  "data": null
}
```

可能的业务错误：

- 用户名已存在
- 邮箱已存在
- 验证码错误
- 验证码已过期
- 验证码不存在或已过期

---

### 6.2 用户登录

- 方法：`POST`
- 路径：`/api/v1/user/login`
- 是否鉴权：否
- Content-Type：`application/json`

#### 请求参数

```json
{
  "username": "xiaowai001",
  "password": "12345678"
}
```

#### 参数说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

#### 成功响应

```json
{
  "code": 0,
  "msg": "登录成功",
  "data": {
    "token": "jwt-token-string"
  }
}
```

#### 失败响应

```json
{
  "code": 401,
  "msg": "用户名或密码错误",
  "data": null
}
```

```json
{
  "code": 429,
  "msg": "登录失败次数过多，请10分钟后再试",
  "data": null
}
```

#### 说明

- 登录成功后，Cookie 中会写入 `token`
- Token 有效期为 7 天
- 连续失败 5 次后限制 10 分钟

### 6.4 获取当前用户资料

- 方法：`GET`
- 路径：`/api/v1/user/profile`
- 是否鉴权：是

#### 请求参数
无

#### 成功响应

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "id": 1,
    "nickname": "小歪",
    "birthday": "2000-01-01T00:00:00Z",
    "avatar": "oss://spaceblue/users/avatar/1+.jpg",
    "gender": "male",
    "hobbies": "音乐,阅读",
    "signature": "今天也要加油",
    "experience": 0,
    "updatedat": "2026-01-01T12:00:00Z"
  }
}
```

#### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| id | number | 用户 ID |
| nickname | string | 昵称 |
| birthday | string/null | 生日 |
| avatar | string | 头像地址 |
| gender | string | 性别 |
| hobbies | string | 爱好 |
| signature | string | 个性签名 |
| experience | number | 经验值 |
| updatedat | string | 更新时间 |

#### 失败响应

```json
{
  "msg": "请先登录"
}
```

或：

```json
{
  "code": 401,
  "msg": "未授权，请重新登录",
  "data": null
}
```

  "msg": "更新成功",
  "data": null
}

### 6.7 预览用户头像

- 方法：`GET`
- 路径：`/api/v1/user/:id/avatar`
- 是否鉴权：否

#### 路径参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | number | 是 | 用户 ID |

#### 成功响应
成功时直接返回图片二进制流，不是 JSON。

响应头示例：

```text
Content-Type: image/jpeg
```

#### 失败响应

```json
{
  "code": 400,
  "msg": "用户ID无效",
  "data": null
}
```

```json
{
  "code": 404,
  "msg": "用户不存在",
  "data": null
}
```

```json
{
  "code": 404,
  "msg": "头像不存在",
  "data": null
}
```

#### 说明
当前默认头像可能不是有效的 OSS 路径，因此新用户未上传头像时，前端应准备默认占位图。

## 9. 常见前端联调问题

### 9.1 为什么登录成功后还是 401？
通常是因为前端没有携带 Cookie：

- `fetch` 缺少 `credentials: 'include'`
- `axios` 缺少 `withCredentials: true`

### 9.2 为什么头像接口返回不是 JSON？
`GET /user/:id/avatar` 返回的是图片流，应该作为 `<img src>` 使用，而不是按 JSON 解析。

### 9.3 为什么首页不能直接判断登录？
当前后端没有单独的“检查登录状态”接口，推荐直接调用：

- `GET /api/v1/user/profile`

成功说明已登录，失败说明未登录。

### 9.4 为什么聊天页面中的历史记录和智能体切换不能真正工作？
因为当前后端尚未提供这些接口，前端现有页面中的相关内容属于展示态或 mock。
