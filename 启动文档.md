# 小歪 Xiaowai 项目启动与部署说明

本文档说明如何启动 `xiaowai-backend`、构建 `xiaowai-web`，以及使用 Nginx 部署整套项目。

## 1. 项目结构

- `xiaowai-backend`：Go 后端服务，默认监听 `8080`
- `xiaowai-web`：React + Vite 前端项目
- `nginx/xiaowai.conf`：Nginx 部署配置
- `docs/`：接口文档、交互流程与前后端联调说明

---

## 2. 前置准备

### 2.1 后端运行环境

请确保服务器或本地环境已安装：

- Go（建议 1.22+）
- MySQL
- Redis
- Nginx

### 2.2 前端运行环境

请确保已安装：

- Node.js 18+
- npm 或 pnpm

---

## 3. 启动后端 `xiaowai-backend`

### 3.1 准备配置文件

后端启动时会优先读取环境变量 `SPACEBLUE_CONFIG` 指向的配置文件；如果未设置，则默认读取：

- `xiaowai-backend/config/config.yaml`

你需要先准备好该配置文件，并正确填写：

- MySQL 连接信息
- Redis 连接信息
- OSS 配置
- 千问模型配置
- SMTP 邮件配置
- JWT 配置
- 日志配置

### 3.2 安装依赖并启动

在 `xiaowai-backend` 目录执行：

```bash
go mod tidy
go run ./cmd/main.go
```

启动成功后，后端服务默认运行在：

```text
http://localhost:8080
```

接口前缀为：

```text
http://localhost:8080/api/v1
```

---

## 4. 启动前端 `xiaowai-web`

### 4.1 安装依赖

在 `xiaowai-web` 目录执行：

```bash
npm install
```

### 4.2 配置开发环境变量

项目已提供示例文件：

- `xiaowai-web/.env.example`

开发时可以创建 `.env` 文件，内容例如：

```env
VITE_API_BASE_URL=http://localhost:8080/api/v1
```

### 4.3 启动开发服务器

```bash
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

---

## 5. 当前前端功能说明

现在的 `xiaowai-web` 已按文档和后端接口完成以下页面：

- 首页：点击“开始使用”会先检查登录态，再决定进入聊天页或登录页
- 登录页：按后端要求使用“用户名 + 密码”登录
- 注册页：支持邮箱验证码、用户名、密码、确认密码、手机号（可选）、协议勾选
- 个人资料页：支持拉取资料、更新资料、上传头像
- 聊天页：支持调用 `/api/v1/user/agent/chat` 进行单轮文本问答

注意：

- 登录态依赖 Cookie，前端请求已默认携带 `credentials: 'include'`
- 聊天接口当前只支持单轮问答
- 历史记录、智能体切换等能力后端暂未提供

---

## 6. 构建前端生产包

在 `xiaowai-web` 目录执行：

```bash
npm run build
```

构建完成后会生成：

```text
xiaowai-web/dist
```

项目已新增生产环境变量文件：

- `xiaowai-web/.env.production`

内容为：

```env
VITE_API_BASE_URL=/api/v1
```

这意味着生产环境中前端会通过当前域名下的 `/api/v1` 访问后端，再交由 Nginx 反向代理。

---

## 7. 使用 Nginx 部署整套项目

### 7.1 复制前端产物

建议将前端构建结果复制到：

```text
/var/www/xiaowai-web/dist
```

例如：

```bash
mkdir -p /var/www/xiaowai-web
cp -r xiaowai-web/dist /var/www/xiaowai-web/
```

如果你不使用这个目录，也可以改动 Nginx 配置中的 `root` 路径。

### 7.2 配置 Nginx

项目提供了配置文件：

- `nginx/xiaowai.conf`

其作用是：

- `/` 提供前端静态资源
- 通过 `try_files` 支持 React Router 刷新不 404
- `/api/` 反向代理到本机 `8080` 后端服务

你可以将该配置复制到：

```text
/etc/nginx/conf.d/xiaowai.conf
```

### 7.3 检查并重载 Nginx

```bash
nginx -t
systemctl reload nginx
```

---

## 8. 生产部署推荐流程

推荐顺序如下：

1. 启动 MySQL、Redis 等基础依赖
2. 启动 `xiaowai-backend`
3. 构建 `xiaowai-web`
4. 将 `dist` 发布到 Nginx 静态目录
5. 放置 `nginx/xiaowai.conf` 并重载 Nginx
6. 浏览器访问域名并测试：
   - 注册
   - 登录
   - 获取资料
   - 上传头像
   - AI 对话

---

## 9. 常见问题

### 9.1 登录成功后接口还是 401

请确认：

- 前端是否通过 Nginx 同域访问
- 后端 Cookie 是否成功写入
- 请求是否携带 Cookie

当前前端已在请求封装中默认加上：

```ts
credentials: 'include'
```

### 9.2 React 路由刷新 404

请确认 Nginx 配置中存在：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 9.3 头像无法显示

后端头像预览接口返回的是图片流，而不是 JSON。当前前端已经按图片地址方式处理：

```text
GET /api/v1/user/:id/avatar
```

如果用户尚未上传头像，前端会显示默认占位头像。

---

## 10. 建议的本地联调方式

### 方式一：前后端分开启动

- 后端：`http://localhost:8080`
- 前端：`http://localhost:5173`

前端 `.env` 配置：

```env
VITE_API_BASE_URL=http://localhost:8080/api/v1
```

### 方式二：走 Nginx 统一入口

- Nginx 对外提供一个域名或端口
- 前端和后端都通过 Nginx 暴露
- 前端生产环境使用：

```env
VITE_API_BASE_URL=/api/v1
```

这种方式更接近正式环境，也更适合处理 Cookie。
